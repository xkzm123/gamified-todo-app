## 二、今天解决的问题

### 问题1：Web 预览空白 → ✅ 已修复
- **原因**：Zustand 库的 `.mjs` 文件包含 `import.meta` 语法，Metro Web 打包后脚本以非 module 方式加载，浏览器报 `Cannot use 'import.meta' outside a module`
- **修复**：直接修补了 `node_modules/zustand/esm/` 下 5 个文件，将 `import.meta.env` 替换为 `window.__DEV__` 检测
- **结果**：http://localhost:8082 可正常预览（8081 端口被旧进程占用）

### 问题2：APK 本地构建内存不足 → 🔄 架构迁移中
- **原因**：Expo + React Native 的 Gradle 构建需要编译大量原生 C++ 模块（react-native-screens, safe-area-context 等），JVM 需要 2GB+ 堆内存，你的电脑空闲 RAM 不够
- **方案**：迁移到 **React + Vite + Capacitor** 架构——APK 只是一个 WebView 壳包裹网页，无需编译原生模块，Gradle 构建极轻量
